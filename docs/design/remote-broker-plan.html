The broker is a thin bridge: it forwards the same WebviewSnapshot the desktop chat already renders to remote clients, and relays their prompts back into the existing Pi RPC controller. No new chat engine — it reuses everything.

What you'd actually do

  1. In VS Code: Pi: Start Remote Session → the broker starts + opens a secure tunnel.
  2. You get a URL + a one-time pairing code (QR shown in a popover).
  3. Open the URL on your phone, enter the code → a token is minted (scoped, expiring).
  4. You now see the live chat. Pick a role:
👁 Spectator
Read-only. Sees messages stream live, tool cards, diffs. Cannot send. Good for watching a long task from the couch.
🎮 Driver
Can send prompts, stop, approve tools. Exactly one driver at a time (lock). Desktop can reclaim control instantly.

Message flow (per prompt)

# Step Channel
1 Phone (driver) sends {prompt:"..."} wss → broker
2 Broker checks token + driver-lock, calls controller.prompt() in-process
3 Pi streams message_update events Pi RPC → controller
4 Controller fires a new WebviewSnapshot in-process
5 Broker broadcasts the snapshot to desktop + all remote clients broker → wss
6 Phone renders the same chat UI (typewriter and all) client

Security model (the hard part)

Concern Mitigation
Exposure No open ports. Outbound tunnel only (VS Code Tunnel uses your GitHub identity, or Cloudflare/ngrok).
Auth One-time pairing code → short-lived signed token (JWT). Per-device revoke. Tokens expire on session end.
Least privilege Spectator can't write. Driver-lock prevents two people fighting. Tool approvals still prompt (inline Allow/Deny).
Scope Broker only exposes the chat protocol — never a shell, never the filesystem directly.
Redaction Reuses the existing log redactor for anything leaving the machine.
Kill switch Status bar item + Pi: Stop Remote Session tears down tunnel + tokens instantly.

⚠ This is the reason it's the biggest item: a networked, authenticated, remotely-drivable coding agent needs its threat model right. Ship spectator-only first, add driver later.
