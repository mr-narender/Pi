# Pi 0.80.10 RPC Coverage Matrix

**Row identity rule:** `C-*`, `E-*`, `U-*`, `X-*`, and `D-*` are unique coverage IDs. Every concrete `piRpc.` action ID below is also unique, even where the action is normally invoked from a view rather than the Command Palette. Automated release validation extracts both columns and rejects duplicates. “Test” names are implementation-ready test case IDs.

**Validated inventory totals:** 34 `C-*` rows, 20 `E-*` rows, 9 `U-*` rows, 19 `X-*` rows, and 8 `D-*` rows = **90 named coverage rows**. The assistant-delta submatrix adds **6** unnamed handling rows, for **96 total table rows**. There are **82** concrete unique `piRpc.` action IDs.

**Machine-checked evidence:** `docs/RPC_COVERAGE_EVIDENCE.json` plus `scripts/validateCoverage.mjs` bind every `C/E/U/X/D` row to a source symbol, command/GUI surface, and automated test name. `npm test` runs the validator first and fails on missing or orphaned references.

Component abbreviations: `RPC` complete client/transport; `SC` session controller/reducer; `CHAT` secure chat webview; `SV` Sessions tree; `OV` Conversation Outline; `QV` Queue tree; `TV` Tool/Workflow tree; `DX` diagnostics; `NAV` file/diff/worktree navigation; `XUI` extension UI broker.

## Startup flags and limitations coverage

Every launch uses argv arrays and always adds `--mode rpc`. Settings tests cover provider/model/thinking/models; all session flags; tools/exclusions; every explicit/disabling resource flag; system/append prompt; verbose; approve/no-approve; offline; executable and environment controls. `--api-key` is intentionally rejected from persisted settings and redacted if supplied transiently.

Non-RPC TUI operations (login/logout, settings picker, resume picker, tree navigation mutation, trust persistence, import/share/reload/hotkeys/changelog/quit, package management) are never sent as built-in slash commands. Appropriate substitutes are terminal launch, documented settings, RPC session/tree readers, returned extension commands, or an explicit “unsupported in RPC” explanation.
